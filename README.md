# AML Alert Auto-Closure System
### Google A2A Agent Architecture

---

## Overview

Three specialised agents collaborate via a lightweight A2A (Agent-to-Agent)
orchestrator to automate AML alert triage:

```
┌─────────────────────────────────────────────────────────────────┐
│                      A2A Orchestrator                           │
│  ┌───────────────┐  ┌────────────────┐  ┌──────────────────┐   │
│  │ TrainingAgent │→ │ AnalysisAgent  │→ │   ReportAgent    │   │
│  │               │  │                │  │                  │   │
│  │ Reads         │  │ Reads          │  │ Writes decisions │   │
│  │ alerts_data   │  │ open_alerts    │  │ back to xlsx +   │   │
│  │ .xlsx →       │  │ _data.xlsx →   │  │ auto_closure_    │   │
│  │ trains LLM    │  │ analyses each  │  │ report.xlsx      │   │
│  │ knowledge     │  │ open alert     │  │                  │   │
│  │ base          │  │ via LLM        │  │                  │   │
│  └───────────────┘  └────────────────┘  └──────────────────┘   │
└─────────────────────────────────────────────────────────────────┘
```

## File Layout

```
aml_a2a_system/
├── main.py                  ← Main entry point (all 3 agents + orchestrator)
```

## Quick Start

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Place your historical alerts file
cp /path/to/alerts_data.xlsx .

# 3a. Generate sample open alerts (for testing)
python generate_open_alerts.py

# 3b. Or copy your real open alerts file
cp /path/to/open_alerts_data.xlsx .

# 4. Run the pipeline
python main.py
```

## Input File Format

Both Excel files must follow this column layout (sheet names are flexible):

| Column       | Type    | Description                              |
|--------------|---------|------------------------------------------|
| Alert ID     | string  | Unique alert identifier                  |
| Customer ID  | string  | Customer reference                       |
| Customer Name| string  | Full name                                |
| Alert Type   | string  | e.g. "AML IB Americas"                  |
| Alert Type ID| integer | Numeric type code                        |
| Score        | integer | Risk score 0–100                         |
| Status       | string  | Open / In Review / Escalated / Closed    |
| Assigned To  | string  | Analyst username                         |
| Created Date | date    | Alert creation date                      |
| Amount       | float   | Transaction amount                       |
| Currency     | string  | ISO currency code                        |
| Country      | string  | ISO 2-letter country code                |
| Description  | string  | Alert narrative                          |
| Priority     | string  | Low / Medium / High / Critical           |

## Output Files

### `auto_closure_report.xlsx`
Three sheets:
- **Summary** — KPI counts (auto-closed, escalated, review)
- **Alert Decisions** — Full per-alert decision table with colour coding
  - 🟢 Green = AUTO_CLOSE
  - 🔴 Red   = ESCALATE
  - 🟡 Amber = REVIEW
- **Knowledge Base** — Closure criteria learned from historical data

### `open_alerts_data.xlsx` (annotated in-place)
Four new columns are appended:
- `Action` — AUTO_CLOSE / ESCALATE / REVIEW
- `Closure Comment` — LLM-generated professional comment
- `Risk Flags` — Detected risk indicators
- `Processed At` — UTC timestamp

Alerts marked AUTO_CLOSE also have their `Status` updated to `Closed`.

## Configuration

Edit the constants at the top of `main.py`:

```python
LLM_ENDPOINT  = "http://your-host:8000/v1/completions"
LLM_MODEL     = "meta-llama/Meta-Llama-3.1-8B-Instruct"
MAX_TOKENS    = 512
TEMPERATURE   = 0.3
```

## A2A Protocol

The orchestrator uses a minimal Task/Message envelope compatible with
Google's Agent-to-Agent (A2A) specification:

- **Task** — carries `input`, `output`, `status`
- **Message** — routes tasks between named agents
- **Shared Knowledge Base** — dict written by Agent 1, read by Agents 2 & 3
- All agents are async; the pipeline runs sequentially by design to ensure
  each agent's output is available before the next starts.
